Linux File System

directories is just a fancy word for folder i mean not really but its best to think of it that way.

directories form a hiearchy or " tree " in which they may contain other files or subfiles within themselves.

Home directories

users personal files often found in home dir. ( EX. cd /home/jack or su jack and cd ( use cd cus when you su from root your normally still in the /root dir) would probally get me into the home dir of the user "jack" or you could use /root to get into the home dir of the user root. you get it by now just home/specified user. )

System dir

this dir normally contains system files, applications, man files or docs, basically everthing. wont really use unless your administrating systems.

programs

bin : normally contains programs

sbin : superuser programs

lib : code for programs

docs

doc : documentation

info : info file about various things

man : manual pages ( Ex. man nmap )

share : info

configuration

etc : config files for system

init.d : config files for linux boot

rc.d : also config for booting

hardware

dev : files for interacting with hardware

media : mount point

mnt : mount point

runtime

var : computer files updated and created while system is running

run : PID contains id's of running processes

tmp : temp file storage

spool : files in transit or waiting to be in transit aka queued
