Nmap Basics

Nmap is a network mapper, it can be used for many things from port scanning to bruteforing port 22 with Nmap scripts. PS. Their is also a gui called Zenmap

Nmap has 3 basic port scan types:

TCP connect scans : (-sT)

SYN "half open" scans : (-sS)

UDP scans : (-sU) I think

Less common port scans :

TCP null scans : (-sN)

TCP fin scans : (-sF)

TCP xmas Scans : (-sX)

Except for UDP scans most work similar however the way they work differs. To understand TCP connect scans you have to understand how a "3 way handshake works" basically it works like this, the attacking machine aka your pc or laptop send a tcp request to the target machine then the target machine sends back SYN/ACK response and the attacking machine sends back a ACK request. Basically it just looks like this :

(Your Machine) sends ~~> SYN (Target machine) then sends you back ~> SYN/ACK and finally you respond (Your machine) sends ~> SYN, Now the 3 way handshake has been completed.

SYN and ACK are flag sets!

If a port is closed when scanning nmap will respond with a Rst aka reset flag, in this way nmap marks that port as closed.

SYN Scans

Syn scans are used to scan TCP port-range of a target, similar to tcp request but work different.

Syn scans are often referred to as "half open" scans or "stealth" scans, Unlike how TCP scans, Syn scans send back a Rst TCP packet.

Nmap Switches

-o (detects operating systems target is running on)

-sV (detects services)

-vv ( their is -v but i have use it this just increases the amount of info nmap will output)

-oN (nmap output will save in normal format)

-A (scans more aggressively than normal)

-t (increases the timing of scan using -t 1-5 higher the level the more "noisy")

-p (just tells nmap to scan specified ports)

-p-(dns) : tells nmap to scan domain

--scipt- : tells nmap to use specified script
